# Решение: Дизайн-система v1 (зафиксировано из 8may2026.html)

**Дата**: 2026-05-08
**Категория**: design
**Статус**: Принято (актуально для MVP-прототипа)

## Контекст
По состоянию на 8 мая 2026 в `artifacts/pages/8may2026.html` собран законченный лендинг + интерактивное демо CRM. Нужно зафиксировать ключевые дизайн-решения, чтобы будущая реализация на React/TypeScript была визуально консистентна с прототипом.

## Решение

### Типографика
- Шрифт: системный sans-serif (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto`)
- Размеры: 10–12px (мета), 13–14px (UI), 15–17px (заголовки секций), 22–32px (H2/H3), 44px (Hero H1)
- Заголовки: `font-weight: 600`, основной текст: 400–500

### Скругления и тени
- Кнопки/инпуты: `border-radius: 8px`
- Карточки: `border-radius: 10–12px`
- Модальные окна: `border-radius: 16px`
- Бейджи/чипсы: `border-radius: 20px`
- Аватары: `border-radius: 50%`
- Тени почти не используются — дизайн «плоский», границы вместо теней

### Иконки
- Библиотека: **Tabler Icons** (`@tabler/icons-webfont`)
- Подключение: CDN или npm-пакет (для prod)

### Бейджи статусов заказа
- `b-new` (Новый) — `#EEEDFE` / `#534AB7`
- `b-work` (В работе) — `#E1F5EE` / `#0F6E56`
- `b-wait` (Ожидание) — `#FAEEDA` / `#854F0B`
- `b-done` (Готово) — `#EAF3DE` / `#3B6D11`

### Адаптивность
- Брейкпоинт: `max-width: 768px` — мобильная вёрстка
- На мобильных:
  - сетки 2–4 колонок схлопываются в 1
  - сайдбар прячется
  - метрики становятся 2-колоночными

## Альтернативы
- **Ant Design** — слишком тяжёлый и узнаваемый, не даёт нужного «лёгкого» ощущения.
- **Material UI** — слишком «гугловый», не вяжется с философией «просто и по-русски».
- **Shadcn UI + Tailwind** — выбран как основной для реализации, т.к. позволяет точно воспроизвести этот стиль с минимумом готовых стилей.

## Последствия
- ✅ При реализации на React+Tailwind легко сохранить визуал
- ✅ Дизайн-токены можно вынести в `tailwind.config.ts`

- 🔧 Тех-долг: нет тёмной темы (зафиксировать в TODO к v2)
