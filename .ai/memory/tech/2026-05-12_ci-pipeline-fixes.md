# [ПАМЯТЬ] Исправление CI Pipeline: Линтинг и E2E тесты

**Дата**: 2026-05-12
**Категория**: tech
**Статус**: Принято

## Контекст

При создании PR в GitHub обнаружились критические ошибки CI pipeline:
1. Ruff I001 - неправильно отсортированные импорты в src/ файлах
2. E2E тесты падали с `ERR_CONNECTION_REFUSED at http://localhost:3000/`
3. Pytest выдавал warning об неподдерживаемой конфигурации `[env]`
4. Отсутствовал отдельный этап проверки качества кода (linting)

## Решение

### 1. Исправлены импорты в src/ файлах
- `src/api/clients.py`: добавлены пустые строки между блоками импортов
- `src/config/settings.py`: исправлен порядок импортов pydantic
- Применено `ruff format` для единообразного форматирования

### 2. Реструктуризован CI pipeline (.github/workflows/e2e.yml)
```yaml
jobs:
  lint:  # Новый job для проверки качества кода
    - ruff check src/ tests/
    - ruff format --check src/ tests/  
    - mypy src/
  
  test:
    needs: lint  # Тесты только после успешного lint
    - Исправлен синтаксис sharding: --shard=${{ matrix.shard }}/2
```

### 3. Исправлена pytest конфигурация
- Удалена неподдерживаемая секция `[tool.pytest.ini_options.env]` из pyproject.toml
- Обновлена конфигурация ruff на новый формат с `[tool.ruff.lint]`

### 4. Исправлены фикстуры и тесты
- Убран автоматический `page.goto()` из фикстуры page в conftest.py
- Добавлены явные `page.goto(settings.base_url)` в smoke тесты
- Исправлены импорты pytest в conftest.py (убран `from pytest import`)

## Альтернативы

## Последствия

- ✅ Плюсы: 
  - CI pipeline стабилен и проходит все проверки
  - Единый стиль кода через ruff
  - Раннее обнаружение проблем через lint job
  - Явные зависимости между job'ами

- ⚠️ Минусы: 
  - Увеличено время выполнения CI (добавлен отдельный lint job)
  - Тесты требуют явного указания URL навигации

- 🔧 Технический долг: 
  - Нужна настройка тестового окружения с веб-сервером для полноценных E2E тестов
  - Рассмотреть добавление pre-commit hooks для локальной проверки
